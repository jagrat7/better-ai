Install Package Plan

bttrai install ai runs a real package install (like npm i ai) and installs the MCP servers + skills the registry maps to that package.

Goal

Make install a thin wrapper over the package manager. Everything after install is one raw argument string that bttrai forwards verbatim to the detected manager's install verb. bttrai also reads the package names out of that string to resolve and install their extras (the "AI dependencies"). install has no flags of its own — every token belongs to the package manager. 

bttrai install ai
# 1) bun add ai        (or npm/pnpm/yarn/deno equivalent)
# 2) installs Vercel AI SDK skills + any ai-matched MCP servers

Flow

1
Capture raw args — take everything after install as a string array, untouched.
2
Resolve package manager — reuse resolvePackageManager (bun / npm / pnpm / yarn / deno) with fallback.
3
Run real install — <manager> <verb> <raw args>, forwarded verbatim (e.g. npm install ai -D).
4
Extract package names — non-flag tokens from the raw args become the deps (strip @version/@tag for matching).
5
Resolve extras — run matcherService.run({ deps }) to find matched MCP servers + skills.
6
Install extras — agents from config favorites; reuse executeInstallations for skills + MCP servers.

CLI Surface

Whatever you type after install is the package manager's command, forwarded verbatim. bttrai only reads the package names from it to add extras.

Command Forwards Extras for
bttrai install ai npm install ai ai
bttrai install ai zod npm install ai zod ai, zod
bttrai install ai -D npm install ai -D ai
bttrai install -D ai npm install -D ai ai
bttrai install ai@5 --save-exact npm install ai@5 --save-exact ai

Registry Change: package-manager install verb

Today src/registry/package-managers.ts only defines a runner (npx/bunx/dlx) for executing installer packages. It has no real install/add verb, so we add one per manager.

Manager Install verb
npm npm install <pkg>
bun bun add <pkg>
pnpm pnpm add <pkg>
yarn yarn add <pkg>
deno deno add npm:<pkg>

Add an installArgs(rawArgs) builder to PackageManagerDefinition that returns [verb, ...rawArgs] (deno prefixes bare package tokens with npm:). bttrai does not interpret or reorder the args — they are forwarded as typed. Add a runInstallCommand util that uses the manager's command (not the runner) via the existing execa wrapper.

Open Decisions

  • Wildcard extras: exclude universal when.deps: ["*"] entries (e.g. context7) from package installs so install ai stays targeted. Offer --with-universal later if needed.
  • No-match packages: if a package has no registry extras, still run the package install and print "no extras found".
  • No bttrai flags on install: the whole arg string belongs to the package manager, so there are no install-specific flags to collide with it. Agent targets for extras are resolved per plan 3 (auto-detect or pinned agents).
  • Package extraction: a token is a package if it does not start with -; strip a trailing @version/@tag before matching. Handles scoped names (@scope/pkg) and flag-first order (-D ai).

Implementation Steps

  1. Wire install in src/cli.ts to capture all args after the command as a raw string[] (variadic positional, flag parsing disabled for this command).
  2. Add installArgs(rawArgs) to each entry in src/registry/package-managers.ts.
  3. Add runInstallCommand(project, manager, rawArgs) in src/services/install/utils.ts (sibling to runPackageCommand).
  4. Add a package-name extractor (non-flag tokens, strip @version) and run matcherService.run on the result.
  5. Create a PackageInstallService (new file under src/services/install/) implementing the 6-step flow; reuse executeInstallations with agents resolved per plan 3.
  6. Handle and surface package-install failures (not found, network) without blocking extras when possible.
  7. Update README usage.

Tests

  • Raw args are forwarded to the manager's install verb verbatim and in order (ai -D, -D ai, ai@5 --save-exact).
  • Package extractor returns only non-flag tokens with versions stripped; handles @scope/pkg and flag-first order.
  • install ai runs the package install then resolves ai extras.
  • Multiple packages install together and merge their extras (de-duped).
  • Wildcard MCP entries are excluded from package-targeted installs.
  • A package with no registry match still installs and reports "no extras".
