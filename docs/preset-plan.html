Preset Plan

Named bundles of MCP servers + skills. Run a preset on its own with the preset command, or layer it on top of another command with --preset.

Part 4 of 4. Presets are stored in the config file defined in plan 3. Presets do not contain agents — agent resolution is owned by plan 3.

Goal

Let users define reusable extras (e.g. a frontend bundle) and either install them on their own or add them on top of another command's results.

bttrai preset frontend            # raw preset only, no detection
bttrai detect --preset frontend   # detected matches + frontend extras

Decisions

  • The preset name command installs only that preset — nothing is detected.
  • The --preset name flag adds preset extras on top of the host command's results.
  • Presets do not contain agents.
  • Presets are independent by default, so there is no includeDetected flag.

Preset Shape

Presets live under presets in the config file.

{
  "presets": {
    "frontend": {
      "description": "Frontend extras",
      "mcp": ["context7", "shadcn-ui"],
      "skills": ["frontend-design", "impeccable-ui"]
    },
    "backend": {
      "description": "Backend extras",
      "mcp": ["postgres"],
      "skills": ["api-design"]
    }
  }
}

Preset Resolution

preset name (command)

Install items = preset.mcp + preset.skills. No detection runs.

--preset name (flag)

Install items = host command results + preset.mcp + preset.skills.

De-dupe merged MCP servers and skills before rendering prompts or installing.

CLI Behavior

CommandResult
bttrai preset frontendFrontend extras only, favorite agents, prompts normally.
bttrai preset frontend --autoFrontend extras only, skips prompts.
bttrai preset frontend --agent cursorFrontend extras only, cursor only.
bttrai detect --preset frontendDetected matches + frontend extras, prompts normally.
bttrai detect --preset frontend --autoDetected matches + frontend extras, skips prompts.
bttrai detect --preset frontend --agent cursorDetected matches + frontend extras, cursor only.

Implementation Steps

  1. Read the presets block from the config schema (defined in plan 3).

  3. Add the --preset option to the detect command.
  4. Merge host command results with --preset extras.
  5. De-dupe MCP servers and skills before prompting/installing.
  6. Validate unknown preset names with a friendly error.
  7. Update README examples and options table.
