Detect Command Plan

Move the project-wide detect-and-install flow into detect, and free up install to mean a package install.

Goal

Today the default command (install) scans the whole project, prompts, and installs every matched MCP server + skill. That whole flow becomes detect. The install name is then reused for package-targeted installs (see plan 2).

Before → After

Before

detect

Scans deps, matches registry, prints MCP servers + skills. Read-only.

install (default)

Scans deps, matches, prompts, installs matched MCP servers + skills to agents.

After

detect (default)

Scans deps, matches, prompts, installs matched MCP servers + skills. Absorbs today's install flow. --list keeps the read-only print mode.

install <package>

Package-targeted. Runs a real package install + its extras. See plan 2.

Command Surface

CommandBehavior
bttraiDefaults to detect: project-wide detect + install.
bttrai detectScan, prompt, install matched MCP servers + skills.
bttrai detect --autoSkip prompts, install resolved matches.
bttrai detect --listPrint matches only, install nothing (today's read-only detect).
bttrai detect --agent cursorInstall to specific agents.
bttrai detect --skills / --mcpScope to one kind.
bttrai detect --jsonMachine-readable output, no execution.

Decisions

  • Default command moves from install to detect.
  • Read-only listing is preserved behind detect --list (or --dry-run); --json already never executes.
  • Flags carry over unchanged: --auto, --agent, --skills, --mcp, --json now live on detect.
  • Preset/config flags (--preset, --add) also attach to detect, not install (plans 3 & 4).
  • Service naming: keep the pure pipeline detectService in src/services/detector/detect.ts; the interactive flow currently in InstallService is repurposed as the detect command handler.

Implementation Steps

  1. In src/cli.ts, make the detect procedure drive the interactive install flow (today's install()), and add its options: auto, agent, skills, mcp, list.
  2. Add a list (read-only) branch that calls the current print logic in detectService.command; default path runs install.
  3. Move default: true from install to detect.
  4. Repurpose InstallService as the detect-flow service (rename to DetectFlowService or keep file, retarget the export) so naming matches behavior.
  5. Reduce the install procedure to a placeholder until plan 2 lands (or land both together).
  6. Drop the --auto requires --agent hard error once config agents exist (coordinated with plan 3).
  7. Update help text, descriptions, and README command table.
